# Walmart Recruiting - Store Sales Forecasting

ეს არის Walmart-ის weekly sales forecasting პროექტი. პროექტში შევადარეთ ოთხი განსხვავებული არქიტექტურა და თითოეული ექსპერიმენტი ცალკე დოკუმენტაციაშია აღწერილი.

## მოდელების შედარებითი ანალიზი

პროექტში გამოვცადეთ 8 განსხვავებული მოდელი, რომლებიც ოთხ ძირითად ოჯახს მოიცავს:

* **Tree-based მოდელები:** CatBoost და LightGBM
* **კლასიკური სტატისტიკური მოდელები:** SARIMA და Prophet
* **Deep Learning მოდელები:** DLinear, N-BEATS და TiDE
* **Transformer-based მოდელი:** TimeXer

ყველა მოდელი შეფასდა Walmart-ის ოფიციალური **Weighted Mean Absolute Error — WMAE** მეტრიკით. ამ მეტრიკაში სადღესასწაულო კვირების შეცდომას ხუთჯერ მეტი წონა აქვს, ამიტომ კარგი მოდელი სწორად უნდა აფასებდეს არა მხოლოდ გაყიდვების ზოგად დონეს, არამედ Thanksgiving-ისა და Christmas-ის მსგავს პერიოდებში წარმოქმნილ მკვეთრ ზრდასაც.

### შედეგების ინტერპრეტაცია

ექსპერიმენტებმა აჩვენა, რომ Walmart-ის მონაცემებისთვის მხოლოდ დროითი სერიის ზოგადი trend-ისა და seasonality-ის სწავლა საკმარისი არ არის. გაყიდვები მნიშვნელოვნად განსხვავდება მაღაზიებისა და დეპარტამენტების მიხედვით, ხოლო holiday კვირებში წარმოიქმნება მკვეთრი და არარეგულარული ზრდა.

**Tree-based მოდელებმა**, განსაკუთრებით CatBoost-მა და LightGBM-მა, კარგად გამოიყენეს Store, Dept, მაღაზიის ტიპი, ზომა, ეკონომიკური მაჩვენებლები, კალენდარული ნიშნები და ისტორიული გაყიდვების lag-ები. მათი მთავარი უპირატესობა იყო განსხვავებული ტიპის feature-ებს შორის რთული არაწრფივი კავშირების სწავლა.

**SARIMA და Prophet** მნიშვნელოვანი კლასიკური baseline-ები აღმოჩნდა. მათი შედეგები მარტივად ასახსნელია და კარგად აჩვენებს trend-სა და seasonality-ს, თუმცა ათასობით Store–Dept სერიის დამოუკიდებლად მოდელირება მათთვის რთულია. ისინი ასევე შედარებით შეზღუდულად იყენებენ სხვადასხვა სერიას შორის საერთო ინფორმაციას.

**DLinear და N-BEATS** აჩვენებს, რომ შედარებით მარტივ deep learning არქიტექტურებსაც შეუძლიათ კონკურენტული პროგნოზების მიღება. DLinear უფრო სწრაფი და მარტივია, ხოლო N-BEATS უკეთ სწავლობს რთულ დროით ფორმებს, თუმცა ორივე მათგანს სჭირდება სწორი scaling, window selection და time-based validation.

**TiDE** განსაკუთრებით საინტერესო აღმოჩნდა, რადგან MLP-ზე დაფუძნებული შედარებით მარტივი არქიტექტურით ერთდროულად იყენებს ისტორიულ გაყიდვებსა და დამატებით covariate-ებს. იგი უფრო მოქნილია, ვიდრე მხოლოდ univariate forecasting მოდელები და Transformer-ებზე ნაკლებ გამოთვლით რესურსს მოითხოვს.

**TimeXer** ყველაზე კომპლექსურ არქიტექტურას წარმოადგენს. მას შეუძლია გრძელი დროითი დამოკიდებულებებისა და exogenous ცვლადების დამუშავება, თუმცა მაღალი კომპლექსურობა ავტომატურად უკეთეს შედეგს არ ნიშნავს. ამ მონაცემთა ზომისა და forecast horizon-ის პირობებში სწორი feature engineering და validation ხშირად უფრო მნიშვნელოვანია, ვიდრე მოდელის არქიტექტურული სირთულე.

## ძირითადი დასკვნები

1. Walmart-ის ამოცანაში **Store–Dept იდენტიფიკატორები და წლიური სეზონურობა** პროგნოზის ყველაზე მნიშვნელოვანი კომპონენტებია.

2. განსაკუთრებით მნიშვნელოვანი აღმოჩნდა **52-კვირიანი lag**, რადგან ერთი წლის წინა შესაბამისი კვირის გაყიდვები ძლიერი seasonal reference-ია.

3. ჩვეულებრივი random split ამ ამოცანისთვის არასწორია. ყველა მოდელი უნდა შეფასდეს **chronological time-based validation-ის** გამოყენებით.

4. მოდელების შერჩევა უნდა მოხდეს უშუალოდ **WMAE-ის მიხედვით**, რადგან ჩვეულებრივი MAE holiday კვირების მაღალ მნიშვნელობას არ ითვალისწინებს.

5. უფრო რთული deep learning ან Transformer არქიტექტურა ყოველთვის არ იძლევა უკეთეს შედეგს. კარგად მომზადებული LightGBM ან CatBoost შეიძლება უფრო სტაბილური იყოს, განსაკუთრებით მაშინ, როდესაც dataset შედარებით მცირეა.

6. კლასიკური მოდელები მნიშვნელოვანია ინტერპრეტაციისა და baseline-ისთვის, მაგრამ მრავალი Store–Dept სერიის ერთდროულად პროგნოზირებისას მათი scalability შეზღუდულია.

7. Local validation და Kaggle score შეიძლება მნიშვნელოვნად განსხვავდებოდეს. ამის ძირითადი მიზეზებია validation პერიოდის არასწორი შერჩევა, leakage, recursive forecasting და train/test distribution-ის განსხვავება.

## საბოლოო შეჯამება

პროექტის შედეგები აჩვენებს, რომ Walmart-ის weekly sales forecasting არის არა მხოლოდ ჩვეულებრივი time-series ამოცანა, არამედ **დიდი რაოდენობის ურთიერთდაკავშირებული Store–Dept სერიების მქონე panel forecasting პრობლემა**.

კლასიკური მოდელები უკეთეს ინტერპრეტაციას გვაძლევს, tree-based მოდელები ეფექტურად იყენებს tabular და lag feature-ებს, ხოლო deep learning მოდელები გვაძლევს შესაძლებლობას ავტომატურად ვისწავლოთ უფრო რთული დროითი დამოკიდებულებები.

პრაქტიკული თვალსაზრისით, ყველაზე სტაბილური მიდგომაა ისეთი მოდელი, რომელიც აერთიანებს:

* Store–Dept-ის სპეციფიკურ ინფორმაციას;
* წლიურ და კალენდარულ სეზონურობას;
* leakage-safe historical lag-ებს;
* holiday feature-ებს;
* დამატებით მახასიათებლებს.

საბოლოოდ, მოდელის არქიტექტურაზე არანაკლებ მნიშვნელოვანი აღმოჩნდა მონაცემების სწორად მომზადება, time-based validation, leakage-ის თავიდან აცილება და Kaggle-ის ოფიციალური WMAE მეტრიკის ზუსტად გამოყენება.
